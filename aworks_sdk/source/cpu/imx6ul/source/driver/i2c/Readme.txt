写驱动时需要确认当前CPU的I2C外设的特性：
1. 将SCL手动接到地，再启动I2C,测试硬件是否能检测总线异常；
2. 将SDA手动接到地，再启动I2C,测试硬件是否能检测总线异常；
3. 将SCL和SDA同时接到地，再启动I2C,测试硬件是否能检测总线异常；


如：在imx6ul中，将SCL拉低后，启动I2C，则I2C会立即进入中断，查询状态寄存器，显示仲裁丢失的错误。
    同理将SDA拉低后，启动I2C后，SCL发送9个时钟后进入中断，显示仲裁丢失的错误。
    
注：I2C驱动的原则是，不管I2C总线发生什么异常程序都能够返回的错误代码，不能一直等待。如果发生致命错误且主机支持复位，应该复位主机。
